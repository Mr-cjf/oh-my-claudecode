# i18n 自动化翻译系统

## 概述

本系统为 fork 项目提供完整的自动化翻译解决方案，能够自动检测上游更新、提取新字符串、并使用 AI 自动翻译。

## 工作流程

```
上游更新 → 检测新字符串 → AI 翻译 → 创建 PR → 人工审核 → 合并
```

## 使用方法

### 1. 手动同步（推荐用于开发）

```bash
# 检测新的硬编码字符串
npm run i18n:sync

# 查看报告
cat .omc/i18n-sync-report.json

# 应用自动翻译
npm run i18n:apply
```

### 2. 自动同步（GitHub Actions）

系统每天自动运行，检测上游更新并创建翻译 PR。

**手动触发**：
1. 进入 GitHub Actions 页面
2. 选择 "i18n Auto-Sync" workflow
3. 点击 "Run workflow"
4. 输入上游仓库和分支（可选）

**自动触发**：
- 每天凌晨 2:00 UTC 自动运行
- 检测到新字符串时自动创建 PR

### 3. Git Hook（本地自动检测）

安装 Git hook 后，每次 pull 上游更新时会自动检测新字符串：

```bash
# 安装 hook
cp hooks/post-merge .git/hooks/
chmod +x .git/hooks/post-merge

# 之后每次 git pull 时会自动检测
git pull upstream main
```

## 配置

### GitHub Actions 配置

在 GitHub 仓库设置中添加 Secret：

1. 进入 Settings → Secrets and variables → Actions
2. 添加 `ANTHROPIC_API_KEY`
   - 名称：`ANTHROPIC_API_KEY`
   - 值：你的 Claude API Key

### 本地环境配置

```bash
# 设置 API Key（用于本地翻译）
export ANTHROPIC_API_KEY=your_api_key_here

# 或在 .env 文件中
echo "ANTHROPIC_API_KEY=your_api_key_here" >> .env
```

## 工作流程详解

### 步骤 1: 检测新字符串

```bash
npm run i18n:sync
```

**功能**：
- 扫描所有 TypeScript 文件
- 提取 `console.log`, `chalk`, `.description()` 中的字符串
- 与现有翻译对比
- 生成报告：`.omc/i18n-sync-report.json`

**输出示例**：
```
🔍 Scanning for hardcoded strings...

Found 125 hardcoded strings
Loaded 89 existing translations

📝 Found 3 new strings to translate:

src/cli/commands/team.ts:
  Line 45: "Team started successfully"
  Line 67: "Agent spawned"

src/hud/render.ts:
  Line 123: "Context limit warning"

🤖 Auto-translating new strings...

  Translating: "Team started successfully"
  ✓ cli.team.started: "团队启动成功"

  Translating: "Agent spawned"
  ✓ cli.team.agentSpawned: "代理已生成"

  Translating: "Context limit warning"
  ✓ hud.context.warning: "上下文限制警告"

📄 Report saved to: .omc/i18n-sync-report.json
```

### 步骤 2: 应用翻译

```bash
npm run i18n:apply
```

**功能**：
- 读取同步报告
- 将新翻译添加到 `en.json` 和 `zh.json`
- 保持 JSON 结构一致

### 步骤 3: 审核和提交

```bash
# 查看变更
git diff src/i18n/locales/

# 测试翻译
npm run build
LANG=zh_CN.UTF-8 node bridge/cli.cjs --help

# 提交
git add src/i18n/locales/
git commit -m "chore(i18n): add 3 new translations"
```

### 翻译质量检查

添加翻译质量验证：

```typescript
function validateTranslation(en: string, zh: string): boolean {
  // 检查长度比例
  if (zh.length < en.length * 0.3) return false;

  // 检查占位符
  const enPlaceholders = en.match(/\{\{[^}]+\}\}/g) || [];
  const zhPlaceholders = zh.match(/\{\{[^}]+\}\}/g) || [];

  if (enPlaceholders.length !== zhPlaceholders.length) return false;

  // 检查是否包含原文
  if (zh.includes(en)) return false;

  return true;
}
```

## 故障排除

### 问题 1: API Key 未设置

**症状**：
```
⚠ ANTHROPIC_API_KEY not set, using placeholder
```

**解决**：
```bash
export ANTHROPIC_API_KEY=your_key_here
```

### 问题 2: 翻译失败

**症状**：
```
⚠ Translation failed: Network error
```

**解决**：
1. 检查网络连接
2. 验证 API Key 是否有效
3. 检查 API 配额

### 问题 3: Git Hook 不执行

**症状**：
`git pull` 后没有检测新字符串

**解决**：
```bash
# 检查 hook 权限
ls -la .git/hooks/post-merge

# 添加执行权限
chmod +x .git/hooks/post-merge

# 手动测试
.git/hooks/post-merge
```

### 问题 4: GitHub Actions PR 创建失败

**症状**：
Workflow 运行但没有创建 PR

**解决**：
1. 检查 `GITHUB_TOKEN` 权限
2. 确认仓库允许 Actions 创建 PR
3. 查看 Actions 日志中的错误信息

```typescript
function applyGlossary(text: string, glossary: Record<string, string>): string {
  let result = text;
  for (const [en, zh] of Object.entries(glossary)) {
    result = result.replace(new RegExp(en, 'gi'), zh);
  }
  return result;
}
```

## 成本估算

### Claude API 成本

- 每个翻译请求：~100 tokens
- 成本：$0.003 / 1K tokens (Claude 3.5 Sonnet)
- 翻译 100 个字符串：~$0.03
- 每月翻译 1000 个字符串：~$0.30

**优化建议**：
- 使用缓存避免重复翻译
- 批量翻译减少 API 调用
- 使用更便宜的模型（Claude 3 Haiku）

async function translateToAllLanguages(text: string): Promise<Record<string, string>> {
